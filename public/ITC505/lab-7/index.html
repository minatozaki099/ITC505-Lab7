Addendum

Development Process

This Mad Lib application was built using ExpressJS to handle HTTP POST requests from an HTML form. The development began by planning a creative story template with six distinct word slots: two adjectives, a plural noun, a verb, a regular noun, and a place. I structured the project with separation of concerns, creating distinct HTML, CSS, and JavaScript files for better maintainability and organization.

Design Choices

I chose a vibrant, playful aesthetic to match the entertaining nature of Mad Libs. The purple gradient background creates visual interest while maintaining readability. The form uses a card-based layout with generous padding and rounded corners to create a modern, approachable interface. Each input field features clear labels, helpful placeholder text with examples, and smooth focus transitions to enhance user experience.

Technical Implementation

The server-side code uses Express's urlencoded middleware to parse form data from POST requests. When users submit the form, the server validates that all fields contain data before processing. If validation fails, users see a styled error page. Upon successful submission, the server performs string interpolation to insert user inputs into the story template and returns a beautifully formatted result page with a link back to the form.
